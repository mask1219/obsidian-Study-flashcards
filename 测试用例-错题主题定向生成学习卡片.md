# 测试用例-错题主题定向生成学习卡片

## 1. 文档说明

本文基于 [PRD-错题主题定向生成学习卡片](</Users/am700/Library/Mobile Documents/iCloud~md~obsidian/Documents/Obsidian知识库/02 个人工作区/产品经理/obsidian学习闪卡/PRD-错题主题定向生成学习卡片.md>) 编写，目标是覆盖以下范围：

- 错题场景下的入口展示
- 本地设置项开关
- 主题识别优先级
- `ai` / `hybrid` 模式下的生成流程
- `rule` 模式下的限制行为
- 单次固定 5 张卡片的生成约束
- 重复卡片跳过
- 中文提示与异常处理
- 新卡字段落地与数据持久化
- 生成后复习视图刷新行为

测试角色统一为插件用户。除特殊说明外，默认插件已正常安装，复习视图可正常打开，且测试用户可访问插件设置页。

## 2. 测试前置

### 2.1 基础测试数据

- 错题卡 A
  - 已在错题本中
  - `sourcePath` 有效
  - `sourceHeading` = `二分查找`
  - `sourceAnchorText` = `查找边界`
  - 问题文本中也包含“二分查找”
- 错题卡 B
  - 已在错题本中
  - 无 `sourceHeading`
  - `sourceAnchorText` = `链表反转`
  - 问题文本包含“链表”
- 错题卡 C
  - 已在错题本中
  - 无 `sourceHeading`
  - 无 `sourceAnchorText`
  - 问题文本可提取核心主题“哈希表”
- 普通卡 D
  - 不在错题本中
- 一组 AI 可正常生成的测试配置
- 一组 AI 调用失败的测试配置
- 一组会触发重复检测的已有卡片数据
- 一篇来源笔记内容完整的测试笔记

### 2.2 通用验证点

每条测试用例执行时，都应同时观察以下内容：

- 页面提示是否为中文
- 提示是否足够明确，能帮助用户判断原因
- 关闭并重新打开复习视图后状态是否仍正确
- 生成后的卡片是否可被现有复习流程读取
- 不相关能力是否未被破坏，例如错题本切换、翻卡、打开原文

## 4. 验收对照

### 4.1 已覆盖的 PRD 重点

- 错题场景下的入口展示规则
- 本地设置项控制入口显示
- 主题识别优先级
- `ai` / `hybrid` 模式可用，`rule` 模式不可用
- 单次固定 5 张卡片
- 重复跳过与中文提示
- 新卡字段 `generatedFromCardId`、`generatedTopic`
- 生成后刷新但停留当前错题
- 失败场景与中文错误提示

### 4.2 建议补充的联调验证

- 不同 AI Provider 下的主题识别与生成一致性
- 大量卡片数据下复习视图刷新性能
- 同一错题多次触发生成后的重复处理稳定性
- 插件重启、Obsidian 重启后的字段兼容性
