# 技术方案-错题主题定向生成学习卡片（实现映射）

## 1. 目标与范围

本方案基于《PRD-错题主题定向生成学习卡片》与《测试用例-错题主题定向生成学习卡片》，将“错题识别 -> 主题提取 -> 定向生成 -> 写库复习”闭环落到现有插件架构。

V1 覆盖范围：

- 仅在复习视图“当前卡片是错题”时展示主题区块
- 新增设置项控制入口展示：`mistakeTopicCardEntryEnabled`
- 主题识别优先级：`sourceHeading > sourceAnchorText > 问题文本核心主题 > AI 补充（ai/hybrid）`
- 仅 `ai` / `hybrid` 可触发生成；`rule` 模式仅提示不可用
- 单次目标生成 5 张，执行重复过滤后可少于 5 张
- 新卡写入来源字段：`generatedFromCardId`、`generatedTopic`（建议含 `generatedFromFlow`）
- 生成后刷新复习队列但保持当前错题停留
- 全链路中文提示，不影响既有复习/错题能力

非范围：

- 多主题候选与手动输入主题
- 跨错题批量生成
- 向量检索去重
- 服务端改造、账号体系、灰度发布

## 2. 数据模型与持久化

## 2.1 设置结构扩展

文件：`src/types.ts`, `src/settings.ts`, `src/pluginSettingsState.ts`

```ts
interface NoteFlashcardsSettings {
  // 已有字段...
  mistakeTopicCardEntryEnabled: boolean;
}
```

默认值建议：`true`（默认开启入口，用户可关闭）。

持久化兼容：

- 老数据未包含该字段时，`loadPersistedSettings` 回落默认值。
- 不影响现有 `aiModelConfigs`、学习步进、复习设置读取。

## 2.2 卡片结构扩展

文件：`src/types.ts`, `src/cardState.ts`（`normalizeCard`）

```ts
interface Flashcard {
  // 已有字段...
  generatedFromFlow?: "mistake-topic";
  generatedFromCardId?: string;
  generatedTopic?: string;
}
```

约束：

- `generatedFromCardId`：V1 必做，值为触发生成的错题卡 `id`
- `generatedTopic`：V1 必做，值为本次最终使用的主题
- `generatedFromFlow`：建议落地，固定 `"mistake-topic"`，便于后续统计

兼容性：

- 老卡片不含以上字段时保持可读，不改变既有行为。

## 4. 核心流程

## 4.1 入口展示流程

1. 读取当前卡片 `currentCard`。  
2. 满足以下条件才显示区块：
   - `settings.mistakeTopicCardEntryEnabled === true`
   - `currentCard.inMistakeBook === true`
3. 若模式为 `rule`：
   - 展示主题（若可识别）与不可用提示
   - 不展示生成按钮
4. 若模式为 `ai/hybrid`：
   - 有主题则展示按钮
   - 无主题则展示空状态提示

## 4.2 主题识别流程

优先级固定：

1. `sourceHeading`（去空白后非空）
2. `sourceAnchorText`
3. 从 `question` 提取核心主题（本地可解释规则）
4. 本地信息不足且模式为 `ai/hybrid` 时，调用 AI 补充识别
5. 仍失败则返回“无可用主题”

本地提取建议（V1）：

- 去除 Markdown 标记、符号和停用词
- 提取长度 2~16 的高价值词组
- 无稳定候选时返回空，由 AI 兜底（仅 ai/hybrid）

## 4.3 生成与写库流程

1. 点击按钮后做前置校验：
   - 当前卡仍在错题本
   - 模式为 `ai/hybrid`
   - 有可用主题
   - 有可用 AI 模型配置
2. 构造错题主题生成请求（目标 5 张）。
3. 解析 AI 返回，转换为 `Flashcard[]`。
4. 为每张卡补齐：
   - `generatedFromCardId = currentCard.id`
   - `generatedTopic = topic`
   - `generatedFromFlow = "mistake-topic"`（若保留）
5. 执行去重写库，得到 `addedCount/skippedCount`。
6. 按结果提示：
   - `added=5, skipped=0`：成功
   - `added>0, skipped>0`：部分重复
   - `added=0`：全部重复
7. 刷新复习队列并保持当前错题位置。

## 5. 去重策略（V1）

轻量规则，命中任一即判重并跳过：

- 题面完全一致（`normalize(question)` 相同）
- 题面+答案完全一致（`normalize(question+answer)` 相同）
- 同 `generatedFromCardId + generatedTopic` 下高相似文本（可选，若实现成本高可延期）

说明：

- V1 不覆盖旧卡，只做新增或跳过。
- 不引入向量检索与阈值配置后台。

## 6. 错误处理与文案规范

必须中文提示，且可定位原因：

- 非错题场景：当前卡片不在错题本，无法触发
- 无主题：当前错题暂未识别到可用主题
- rule 模式：该能力需要可用的 AI 生成能力
- AI 配置缺失：当前未配置可用 AI 模型
- AI 请求失败：明确提示请求失败（鉴权/限流/网络/服务异常）
- 写库失败：卡片已生成，但写入本地卡片库失败，请稍后重试
- 全部重复：当前主题相关卡片已存在，本次未新增卡片

## 9. 交付与回滚

交付标准：

- PRD 验收项与 TC01-TC24 全部可追溯到实现点
- 不破坏已有“生成当前笔记/文件夹、复习筛选、错题本、已掌握”流程

回滚策略：

- 设置开关可一键关闭入口（`mistakeTopicCardEntryEnabled=false`）
- 即使关闭入口，已生成卡片仍可正常复习，不影响数据兼容
